## Reporting a Vulnerability

We take security seriously. If you discover a security vulnerability, please follow these steps:

### Where to Report

**DO NOT** open a public issue for security vulnerabilities.

Instead, please report security issues by:

1. Opening a private security advisory on GitHub
2. Or by creating a new issue labeled "security" with minimal details and requesting private communication

### What to Include

When reporting a vulnerability, please include:

* **Description** of the vulnerability
* **Steps to reproduce** the issue
* **Potential impact** of the vulnerability
* **Suggested fix** (if you have one)
* **Your contact information** for follow-up

### Response Timeline

* **Initial Response**: Within 48 hours
* **Status Update**: Within 5 business days
* **Resolution Timeline**: Varies based on severity, but we aim to address critical issues within 30 days

### What to Expect

1. We will acknowledge receipt of your vulnerability report
2. We will investigate and validate the issue
3. We will work on a fix and keep you updated on progress
4. Once resolved, we will notify you and publicly acknowledge your contribution (unless you prefer to remain anonymous)

## Security Best Practices

For users of this repository:

* Keep your dependencies up to date
* Use strong authentication methods
* Follow GitHub's security best practices
* Enable two-factor authentication on your GitHub account
* Review and understand code before running it

## Security Features

This repository includes:

* **Dependabot** for automated dependency updates
* **Branch protection** rules (recommended)
* **Code scanning** via GitHub Actions (when applicable)
* **MIT License** for clear usage terms
